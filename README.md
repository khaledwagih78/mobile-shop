# خالد لقطع غيار المحمول — نظام المبيعات والمخزون

نظام ERP يعمل **بدون إنترنت بالكامل** (Offline-first PWA) — موبايل + كمبيوتر + ويب من كود واحد.

## ✅ الموجود في النسخة الأولى (v1)

| الوحدة | الحالة |
|---|---|
| فاتورة بيع (بحث بالاسم/الكود/الباركود، خصم، آجل، طباعة، واتساب) | ✅ |
| فاتورة شراء (تحديث المخزون والتكلفة تلقائياً) | ✅ |
| المخزون (كود، ماركة، نوع، حد أدنى، تنبيه نواقص، حركة الصنف) | ✅ |
| العملاء والموردين (رصيد، كشف حساب، تحصيل دفعات) | ✅ |
| الأرباح التلقائية (بيع − تكلفة) لكل فاتورة وصنف وعميل | ✅ |
| التقارير (يومي، شهري، أرباح أصناف/عملاء، الأكثر مبيعاً، حركة مخزون، ديون) | ✅ |
| المصروفات + الربح النهائي بعد المصروفات | ✅ |
| المستخدمين والصلاحيات (مدير / مبيعات / مخزن) | ✅ |
| إلغاء فاتورة بصلاحية المدير (مع عكس المخزون والأرصدة) | ✅ |
| نسخ احتياطي مشفّر (AES-256) + استرجاع + نقل لجهاز جديد | ✅ |
| عمل Offline كامل + تثبيت كتطبيق (PWA) | ✅ |
| قائمة انتظار المزامنة السحابية (Supabase — المرحلة 2) | 🟡 جاهزة البنية |

**الدخول لأول مرة:** المستخدم "المدير" — الرقم السري `1234` (غيّره من شاشة المستخدمين).

## 🚀 التشغيل محلياً

```bash
npm install
npm run dev
```

## 🌐 النشر على Netlify (الأسهل)

الطريقة 1 — سحب وإفلات:
```bash
npm run build
```
ثم اسحب مجلد `dist` إلى موقع Netlify Drop.

الطريقة 2 — عبر GitHub: ارفع المشروع، واربطه بـ Netlify مع:
- Build command: `npm run build`
- Publish directory: `dist`

بعد النشر: افتح الرابط من موبايل أي موظف → "إضافة إلى الشاشة الرئيسية" → يشتغل كتطبيق حتى بدون نت.

## 🏗️ المعمارية

- **React 18 + Vite** — واجهة سريعة
- **Dexie (IndexedDB)** — قاعدة بيانات محلية داخل الجهاز، كل عملية تُحفظ فوراً Offline
- **vite-plugin-pwa** — تثبيت كتطبيق + عمل بدون إنترنت
- **Web Crypto AES-GCM** — تشفير النسخ الاحتياطية
- **syncQueue** — كل عملية كتابة تُسجَّل في طابور جاهز للرفع إلى Supabase في المرحلة الثانية
